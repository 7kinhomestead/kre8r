What It Is

The AI Editor

SelectsΩr v2 is the editing intelligence at the core of EditΩr. It takes everything VaultΩr built — classified clips, transcripts, project links — and makes editorial decisions: which talking-head takes are keepers, what order they go in, and where b-roll should cover gaps. The result is a rough cut in seconds, not hours.

Three shoot modes handle every type of shoot. A scripted tutorial gets matched beat-by-beat against the WritΩr script. A freeform documentary where the creator went off-script gets reasoned about from the footage itself. A hybrid sits in between — script-guided but flexible enough to capture the moments that deviated from the plan and were better for it.

The editorial logic is built on a decision gate: every clip is classified by shot_type before any selection logic runs. Talking-head clips go through transcript alignment. B-roll gets flagged for coverage suggestions. Unusable gets filtered out entirely. Nothing ambiguous reaches the selection engine.

Three Shoot Modes

One Engine, Three Contexts

Mode 01
Scripted
Each beat from the WritΩr script matched to the best available talking-head clip by transcript alignment. Highest editorial precision.
Mode 02
Hybrid
Script-guided but flexible. Follows the structure while leaving room for off-script moments that are worth keeping as-is.
Mode 03
Freeform
No script. Claude reasons across all clips to find narrative order from the footage itself. Best for documentary-style content.
How It Works

Step by Step

1
Select Project + Mode
Select the active project. All VaultΩr footage linked to that project loads automatically. Creator selects the shoot mode: SCRIPTED, HYBRID, or FREEFORM.
2
Decision Gate Fires
Every clip is classified by shot_type before any selection logic runs. Talking-head clips go to the alignment engine. B-roll queued for coverage. Unusable filtered out. No ambiguity reaches the selector.
3
SCRIPTED: Beat-by-Beat Matching
Each WritΩr beat is matched to the best talking-head clip by transcript text alignment. Claude reads both the intended script and the actual delivered transcript — picks the strongest take per beat.
4
FREEFORM: Narrative Reasoning
With no script as reference, Claude reads all transcript text across the full footage set and reasons about the best narrative order — finding the thread in the footage rather than mapping footage to a plan.
5
B-Roll Suggestions Generated
Coverage gaps in the rough cut — moments where no good talking-head take exists — are flagged. VaultΩr's b-roll library is searched for clips that fit the topic and timing.
6
Rough Cut Stored
The full selects sequence stored in the selects table — footage_id, order, beat_id, include flag, and notes for every entry. No new files created.
7
Open ReviewΩr →
Handoff button sends the creator to ReviewΩr. The rough cut sequence is waiting there — ready for human approval, CutΩr analysis, and clip extraction decisions.
Technical notes: Accepts both talking-head (hyphen) and talking_head (underscore) — normalized at VaultΩr intake. Confidence check removed in v2 — routing is purely on shot_type. Proxy path must be set on the VaultΩr record before transcription can run.
