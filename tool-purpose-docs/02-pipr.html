What It Is

The Project Brain

Before PipΩr, your idea is just a creative direction. After PipΩr, it's a real project — with an ID in the database, a story structure, and a beat map that every other tool in the pipeline will hang off. This is the moment the video goes from concept to commitment.

The core job is the beat map. You pick a story framework — Save the Cat, Story Circle, VSL (Video Sales Letter), or Freeform — and Claude generates a timed, labeled breakdown of the video's emotional arc. Not just a list of sections. Actual beats: the moment of tension, the pivot, the payoff, the call to action. Each beat has a name, an emotional function description, and an approximate duration.

That beat map is what makes the rest of the pipeline coherent. WritΩr writes to it. DirectΩr shoots to it. EditΩr cuts to it. Without PipΩr, every tool downstream is guessing at structure. With it, the whole video has a spine before a single word is written or a frame is shot.

Save the Cat
Hollywood's 15-beat structure. Built for story tension and emotional payoff. Best for narrative-driven content.
Story Circle
Dan Harmon's 8-step loop: you → need → go → search → find → take → return → change. Great for transformation arcs.
VSL
Video Sales Letter structure. Hook → problem → agitate → solution → proof → offer → CTA. Best for your signature show offers and community content.
Freeform
No template. Claude infers structure from the content. Best for day-in-the-life and unscripted formats.
How It Works

The Workflow

1
Name the project and confirm the video topic
If arriving from Id8Ωr, the Vision Brief pre-populates topic, angle, and title options. If starting fresh, these are entered manually.
2
Select a story framework
Choice is stored with the project record. Framework selection determines the beat template Claude works from.
3
Claude generates the beat map — named beats with emotional functions and timestamps
callClaude() in src/utils/claude.js with the selected framework template + Vision Brief context. Output is structured JSON: beat_name, emotional_function, order, duration_estimate.
4
Review and confirm the structure
Beats can be adjusted before confirmation. Once locked, the beat map becomes the structural contract for every downstream tool.
5
Project record written to the database — project_id issued
First DB write of the pipeline. projects table row created with title, topic, angle, youtube_url placeholder, and state = 'pre-production'.
6
Beat rows written to DB — project_id links all beats to this project
beats table rows created. Each beat: beat_name, emotional_function, order, duration_estimate. WritΩr queries these by project_id at script generation time.

Valuable Final Product

What You Walk Away With

The Beat Map — the video's emotional skeleton. Everything downstream writes to it, shoots to it, cuts to it.

A confirmed project in the database with a named story structure and timed beats — each one carrying its emotional function as plain English. No guessing at intent when you're on set or in the edit.

  • WritΩr generates script segments beat by beat — so the script matches the structure instead of the structure being reverse-engineered from the script.
  • DirectΩr turns the beat map into a shot list — every beat becomes a camera directive, not a vague instruction.
  • EditΩr uses the beat map as the editing frame — rough cut assembly starts from structure, not instinct.
