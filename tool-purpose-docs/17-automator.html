What It Is

The Automation Layer

Kajabi's API doesn't support broadcasts, sequences, or community posts. Every email MailΩr generates has to be manually copied and pasted into Kajabi's web interface — subject line, body, tier selection, scheduling. For 9 emails across three tiers, that's 9 copy-paste sequences done by hand, every video. AutomatΩr is being built to end that.

Using Playwright browser automation, AutomatΩr controls Chromium to open Kajabi, navigate to broadcasts, fill in the content MailΩr wrote, select the correct recipient segment based on AudiencΩr data, and schedule or publish — without the creator touching the keyboard. The creator clicks "Send to Kajabi" in MailΩr. AutomatΩr handles everything from there.

It's the bridge between AI-generated content and actual audience delivery. Without it, the creator is the bridge. Once it works, the pipeline is complete from idea to audience — no human in the loop for the distribution layer unless they want to be.

How It Works

Planned Workflow

1
Creator Clicks "Send to Kajabi" in MailΩr
Trigger passes the project_id, target tier, and email day to AutomatΩr. Creator sets it and walks away.
2
AutomatΩr Launches Playwright
Chromium browser launches server-side. Session cookies persisted from a prior login — avoids triggering Kajabi's CAPTCHA on every run.
3
Navigate to Kajabi Broadcasts
Playwright navigates to the correct Kajabi product's broadcast section. Segment selection applied based on tier (free / paid / inner circle) using tag data from AudiencΩr.
4
Fill Subject, Body, Recipient Segment
Subject line and body from MailΩr's emails table fill the Kajabi broadcast form. Recipient segment set to the correct tier tag.
5
Schedule or Send
Based on creator instruction: send immediately or schedule for the correct day offset (Day 0 today, Day 3 three days out, Day 7 seven days out). Kajabi scheduling UI used directly.
6
Confirmation Returned
Playwright extracts Kajabi broadcast ID from the confirmation page. Written to automator_runs: action_type, target_platform, status, kajabi_id, timestamp. Creator sees confirmation in MailΩr UI.
Playwright + Chromium required Windows only (current architecture) Kajabi session persistence CAPTCHA awareness
