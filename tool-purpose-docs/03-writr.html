What It Is

The Voice Machine

Most AI writing tools produce content that sounds like AI wrote it — competent, flat, nobody in particular. WritΩr is different because it's calibrated to a specific human. Before WritΩr generates a single word, it loads a voice profile: a deep analysis of how the creator actually talks. How they open. Where they put the jokes. The rhythm of their sentences. The places they go off-script. Voice profiles are built from real videos and live in the voice library.

The script is written to the beat map PipΩr created. Not as one long block of text — beat by beat, with each beat's emotional function serving as the creative directive. Beat 3 says "Catalyst — the moment everything went sideways." WritΩr writes that moment. The structure comes from PipΩr; the voice comes from the voice profiles; the words belong to the creator.

Three modes let you control how much WritΩr does versus how much the creator ad-libs on camera: Full Script (every word written out), Bullets (talking points only — trust the talent), or Hybrid (scripted intro and outro with bullet body for the middle). A voice blend slider lets you mix tones — more conversational, more educational, whatever the video calls for.

How It Works

The Workflow

1
Load a project — beat map loads automatically
WritΩr queries the beats table by project_id. All beat_name, emotional_function, and duration_estimate values load into beat cards displayed in the UI. Vision Brief context also loads.
2
Select script mode: Full Script, Bullets, or Hybrid
Mode selection changes the Claude prompt structure — Full Script requests fully formed prose; Bullets requests concise talking points; Hybrid applies mode logic per beat position (first/last beats = scripted, middle = bullets).
3
Adjust the voice blend slider
0–100 scale blending between two selected voice profiles from the library. Slider value passed into the Claude prompt as a weighting instruction — e.g. "70% conversational, 30% educational".
4
Claude writes to each beat using the selected voice profile
callClaude() runs once per beat or as a single batched generation depending on script length. Voice profile analysis text is injected into the system prompt. Emotional function description is the per-beat creative directive.
5
Beat cards display — each shows the beat name, emotional function, and generated script
UI renders one card per beat. emotional_function shown in dimmed italic above the script output so the intent is always visible while editing.
6
Creator reviews and edits inline — then approves
Inline editing before approval. Once approved: writr_scripts row updated with is_approved = true. Approved script becomes the canonical source for downstream tools.
7
Approved script stored in DB — accessible to all downstream tools
full_script, generated_script, voice_profile_id, is_approved, project_id all written to writr_scripts. PackageΩr, CaptionΩr, MailΩr query the approved script as creative context.
