What It Is

The Broadcast Engine

Every video deserves more than one email. The day it drops, you need people to watch it. Three days later, you can give them something useful from inside the video. Seven days in, you invite them to go deeper — the community, the offer, the next level. MailΩr writes all three moments, for all three community tiers, in a single generation.

The three tiers map to your community structure on Kajabi: the free tier, the paid tier, and the inner circle tier. Each tier gets its own version of each email — same voice, different depth, different CTA. The inner circle email feels like something written just for them, because it is.

That's up to 9 emails plus an optional blog post and community post — all written in the creator's voice, informed by the selected content package — in one session. The creator reviews and copies each piece into Kajabi manually, because Kajabi's API doesn't support broadcasts. That last step is what AutomatΩr is being built to eliminate.

How It Works

Step by Step

1
Load Project
Selected package, video URL, and caption context load automatically. The package title, hook, and angle inject into the system prompt as tone anchor before any email is written.
2
Toggle Tiers and Optional Content
Creator selects which tiers to write for (free tier / paid tier / inner circle tier). Toggles for blog post and community post. Optional: next video tease, live offer, community event — these inject into the relevant tier emails.
3
Claude Generates the Full Sequence
Day 0 everyone email + Day 3 per tier + Day 7 per tier. Each email: subject line + full body in plain text (no markdown). Token budget 8192. jsonrepair handles malformed JSON from Claude (literal newlines in email bodies). Stop reason detected before JSON parse.
4
Creator Reviews and Copies
All emails displayed with copy buttons. Creator copies subject + body into Kajabi's broadcast tool for each tier and schedules. No automated send — Kajabi API has no broadcast endpoint.
5
Emails Stored in Database
Written to emails table: project_id, day, tier, subject, body. Blog content and community post content stored per project. Content persists until manually deleted or regenerated.
Kajabi Limitation
Kajabi's Public API has no broadcast, sequence, or community post endpoint. All email content generated by MailΩr must be manually copied into Kajabi's web interface. AutomatΩr is being built to automate this step via Playwright browser automation.
